# PL-20260717: Adaptive Agents Markdown Browser

- Status: Backlog
- Readiness: Research
- Created: 2026-07-17
- Tags: ui, markdown, project-layer, tooling

## Objective

Build a standalone, browser-based markdown browser that lives in the Project Layer. It serves markdown from the repo root with proper link following, image rendering, back/forward navigation, and real-time reactivity to file changes — no more relying on clunky IDE preview.

## Problem Spec

The IDE's built-in markdown preview is sufficient for a quick glance but has no meaningful navigation: following a link opens the linked file but there is no back button, no forward button, no history. Browsing a set of related documents requires opening new tabs or clicking around and losing your place. Images, cross-references, and directory browsing are all clunky or absent.

Users spend extensive time in the IDE working with markdown documentation (instructions, plans, retrospectives, playbooks). A dedicated browser that understands the repo's directory structure, renders markdown faithfully, and lets the user navigate freely would dramatically improve the experience.

The MVP must be simple, zero-infrastructure, and live within the Project Layer so it's always available alongside the guidance it renders.

## Design Decisions (Pre-Activation)

| Decision | Outcome |
|---|---|
| Architecture | Python HTTP server + static JS frontend. One dependency: `watchdog` for OS-native file notifications. |
| Server framework | Python stdlib `http.server` + threading. No Flask — keeps the dependency footprint minimal. |
| Reactivity | `watchdog.Observer` on a background thread feeds events to an SSE endpoint (`/events`). Browser receives push notifications when files change — no polling, no manual refresh. |
| Markdown rendering | `marked.js` from CDN. Well-tested, fast, widely used. Can be bundled for offline use later. |
| Navigation | History API (`pushState` + `popstate`). Back/forward work natively. |
| File serving | Server runs from the repo root so relative links like `../instructions/coding.instructions.md` and `![](images/foo.png)` resolve correctly. |
| App structure | `.adaptive-agents/ui/index.html` (shell) + `.adaptive-agents/ui/app.js` (logic). Generated by the script. |
| Change notification scope | SSE pushes `file_changed`, `file_added`, `file_removed`, `tree_changed` events. The browser re-renders the current file if it changed, refreshes the sidebar tree on structural changes. |

## Scope

1. Create `.adaptive-agents/scripts/ui.py` with subcommands:
   - `generate` — produces `index.html` and `app.js` from embedded templates.
   - `serve` — starts the HTTP server (auto-generates if output missing).
   - The script handles dependency checks (watchdog installed, Python version) with clear error messages.
2. Implement the HTTP server (`py -3 .adaptive-agents/scripts/ui.py serve`):
   - Serves `.adaptive-agents/ui/index.html` as the app entrypoint at `/`.
   - Serves repo files at their actual paths (`/instructions/coding.instructions.md`, etc.).
   - Provides `/api/tree` — JSON directory tree of markdown files and directories.
   - Provides `/events` — SSE endpoint for file-change notifications.
   - Provides `/api/file?path=...` — returns raw file content with Content-Type detection (markdown, image, etc.).
3. Implement the frontend (`index.html` + `app.js`):
   - Left sidebar: file tree loaded from `/api/tree`, clickable to navigate.
   - Right pane: rendered markdown via `marked.js` (CDN).
   - History API for back/forward navigation.
   - `.md` links navigate within the app; external links open in new tab; images render inline.
   - SSE connection to `/events` for reactivity: re-render current file on change, refresh tree on structural changes.
4. File watching with `watchdog`:
   - `watchdog.Observer` monitors the repo root recursively on a background thread.
   - Events debounced (300ms) to avoid thrashing on bulk saves.
   - Changes pushed to SSE clients via a shared queue.

## Dependencies

- Python 3.9+ (watchdog requirement)
- `watchdog` 6.x — `pip install watchdog` (pure Python, no compiled extensions)
- `marked.js` — loaded from CDN (no npm dependency)

## Out of Scope (MVP)

- Fancy dashboard, charts, or widgets. This is a markdown browser.
- Bundle `marked.js` for offline use — that's a future iteration.
- WebSocket or bidirectional communication — SSE is sufficient for file-change push.
- File editing, creation, or deletion through the UI. Read-only browser for MVP.
- Dark mode / theme switching — MVP uses system preference via `prefers-color-scheme`.
- Search — out of scope for now, easy to add later via the `/api/tree` index.
- This work unit is scoped to the Project Layer only. Future iterations may graduate to the canonical `ui/` path for use by multiple Project Layers.
